# 手順書 11：Google Analytics 4（GA4）の設定

## 概要
サイトのアクセス計測を開始する。誰がどのページをどう見たかを確認できる。

---

## Step 1：GA4 プロパティを作る

1. https://analytics.google.com/ を開く（Googleアカウントでログイン）
2. 左下の「⚙ 管理」→「＋ プロパティを作成」
3. プロパティ名（例：和香松庵 Official HP）を入力 → 次へ
4. 業種「芸術とエンターテインメント」→ 次へ
5. ウェブを選択 → サイトURL に `https://wakamatsu-iori.com` → ストリーム作成
6. 「測定ID」をコピーする（`G-XXXXXXXXXX` 形式）

---

## Step 2：測定IDを設定する

> **重要：このサイトは GitHub Actions が `npm run build` でビルドし、出来上がった `dist/` を
> Cloudflare Pages へ `wrangler pages deploy` でアップする構成（`.github/workflows/build.yml`）。**
> Astro は静的サイトなので `PUBLIC_GA4_ID` は **ビルド時（＝GitHub Actions側）** に必要。
> **Cloudflare Pages 管理画面の環境変数は、このビルドには効かない**（Pagesは完成品を配るだけ）。

### 本番（GitHub Actions のビルドに渡す）
GA4測定IDはクライアントHTMLに出る**公開ID（秘密ではない）**なので、`build.yml` のビルド手順の
`env:` に直接書いている：
```yaml
- name: ビルド…
  run: npm run build
  env:
    ICS_URL: ${{ secrets.ICS_URL }}
    YOUTUBE_API_KEY: ${{ secrets.YOUTUBE_API_KEY }}
    PUBLIC_GA4_ID: G-XXXXXXXXXX   # ← ここに測定IDを入れる
```
ID を変えたい/外したいときは、この行を書き換えて push すれば次のビルドで反映される。

### ローカル確認用（任意・.env）
ローカルで `npm run dev` してタグ出力を確認したいときだけ `.env` に追加（gitには乗らない）：
```
PUBLIC_GA4_ID=G-XXXXXXXXXX
```
※ローカルでブラウザ表示すると自分のアクセスもGAに計測される点に注意。

---

## Step 3：Search Console と連携する

1. https://search.google.com/search-console/ を開く
2. 「プロパティを追加」→ URL プレフィックス → `https://wakamatsu-iori.com`
3. 確認方法「Google アナリティクス」を選択（GA4 と同じ Google アカウントなら自動確認）
4. サイトマップを送信：`https://wakamatsu-iori.com/sitemap.xml`

---

## 補足

- GA4 の計測データが反映されるまで最大48時間かかる場合がある
- `PUBLIC_GA4_ID` が空（未設定）のときは GA スクリプト自体が HTML に出力されない設計になっている（計測ゼロ＝プライバシー的に安全なフォールバック）
