# 手順書 11：Google Analytics 4（GA4）の設定

## 概要
サイトのアクセス計測を開始する。誰がどのページをどう見たかを確認できる。

---

## Step 1：GA4 プロパティを作る

1. https://analytics.google.com/ を開く（Googleアカウントでログイン）
2. 左下の「⚙ 管理」→「＋ プロパティを作成」
3. プロパティ名（例：和香松庵 Official HP）を入力 → 次へ
4. 業種「芸術とエンターテインメント」→ 次へ
5. ウェブを選択 → サイトURL に `https://wakamatsu-iori.com` → ストリーム作成
6. 「測定ID」をコピーする（`G-XXXXXXXXXX` 形式）

---

## Step 2：測定IDを設定する

### ローカル確認用（.env）
`.env` に以下を追加：
```
PUBLIC_GA4_ID=G-XXXXXXXXXX
```

### 本番（GitHub Secrets → Cloudflare Pages 環境変数）
Cloudflare Pages の管理画面 → 「設定」→「環境変数」に追加：
- 変数名：`PUBLIC_GA4_ID`
- 値：`G-XXXXXXXXXX`

> **なぜ2箇所？**  
> ローカルは `.env` で管理、本番の Cloudflare Pages は管理画面の環境変数で管理。
> GitHub Secrets は `npm run build` を実行する GitHub Actions 側の環境なので、
> Astro の `PUBLIC_` 変数を Cloudflare に渡すには Pages 側の環境変数設定が必要。

---

## Step 3：Search Console と連携する

1. https://search.google.com/search-console/ を開く
2. 「プロパティを追加」→ URL プレフィックス → `https://wakamatsu-iori.com`
3. 確認方法「Google アナリティクス」を選択（GA4 と同じ Google アカウントなら自動確認）
4. サイトマップを送信：`https://wakamatsu-iori.com/sitemap.xml`

---

## 補足

- GA4 の計測データが反映されるまで最大48時間かかる場合がある
- `PUBLIC_GA4_ID` が空（未設定）のときは GA スクリプト自体が HTML に出力されない設計になっている（計測ゼロ＝プライバシー的に安全なフォールバック）
