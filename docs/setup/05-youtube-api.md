# 手順書05: YouTube Data APIキーとチャンネルIDを用意する

所要時間: 約15分。PCのブラウザで行ってください。

## なぜ必要？

「動画」ページの自動掲載（最新動画・再生リスト別のまとめ）と、トップの「配信中」バッジに使います。
YouTubeの公式API（YouTube Data API v3）を読み取り専用で使うための **APIキー** と、対象の **チャンネルID** が必要です。
どちらも無料です（APIは無料枠内で十分。後述）。

## 手順

### 1. Google Cloud でプロジェクトを作る

1. PCで [Google Cloud Console](https://console.cloud.google.com/) を開く（Googleアカウントでログイン）
2. 画面上部のプロジェクト選択 → **新しいプロジェクト** → 名前は「vtuber-hp」などでOK → **作成**
3. 作成したプロジェクトが選ばれていることを確認

### 2. YouTube Data API v3 を有効化する

1. 左上メニュー → **APIとサービス** → **ライブラリ**
2. 検索欄に「**YouTube Data API v3**」と入力して開く
3. **有効にする** をクリック

### 3. APIキーを作る

1. **APIとサービス** → **認証情報** を開く
2. 上部の **＋ 認証情報を作成** → **APIキー** をクリック
3. 表示されたキー（`AIza...` のような文字列）をコピーしておく

### 4. （推奨）APIキーに制限をかける

漏れても被害を小さくするため、キーの使い道を絞ります。

1. 作成したAPIキーの **編集**（鉛筆アイコン）を開く
2. 「**APIの制限**」→ **キーを制限** → 一覧から「**YouTube Data API v3**」だけにチェック → **保存**
3. （「アプリケーションの制限」はサーバーから使うため「なし」のままでOK）

### 5. チャンネルIDを調べる（UCで始まる文字列）

1. PCで [YouTube Studio](https://studio.youtube.com/) を開く
2. 左下の **設定** → **チャンネル** → **詳細設定**
3. 「**チャンネルID**」（`UC` で始まる24文字）をコピー
   - ※ `@ハンドル名` ではなく、`UC...` の方をコピーしてください

### 6. プロジェクトに設定する

**APIキー（秘密）** と **チャンネルID（公開情報）** で置き場所が違います。

- **APIキー** → `.env` ファイルに設定（ローカル開発用）
  ```
  YOUTUBE_API_KEY=AIzaここに貼り付け
  ```
  ※ `.env` は `.gitignore` 済みなのでコミットされません。

- **APIキー** → GitHub Secrets にも登録（本番の自動ビルド用）
  1. GitHubのリポジトリ → **Settings** → 左メニュー **Secrets and variables** → **Actions**
  2. **New repository secret** → Name: `YOUTUBE_API_KEY` / Secret: コピーしたキー → **Add secret**

- **チャンネルID** → `src/config/site.ts` の `sources.youtubeChannelId` に貼り付け
  ```ts
  sources: {
    youtubeChannelId: 'UCここに貼り付け',
    ...
  }
  ```

### 7. 動作テスト

1. ターミナルで `npm run dev` を実行し、表示URL（通常 http://localhost:4321）を開く
2. 「動画」ページ（/videos）に最新動画・再生リストが表示されればOK
3. トップ（/）の「最新動画」にも数件表示されます

## クォータ（使用量）について

- 無料枠は **1日あたり1万ユニット**。
- このHPは1回のビルドで概ね **十数ユニット** しか使いません（高コストな検索APIは使わない設計）。
- 15分ごとに自動ビルドしても1日あたり約1,000〜1,500ユニットで、上限に十分余裕があります。

## 注意

- **APIキーは絶対にコミットしない**（`.env` とGitHub Secretsだけに置く）。
- 「動画が表示されない」場合は、チャンネルIDが `UC...` になっているか、APIが有効化されているかを確認してください。
- 再生リストは **YouTube側で公開** のものだけがミラーされます（非公開・限定公開は出ません）。仕分けはYouTubeの再生リスト管理だけで完結します。
